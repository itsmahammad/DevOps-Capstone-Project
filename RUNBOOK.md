## Deployment

### Prerequisites
- GitHub `production` environment is protected with required reviewers
- GitHub secrets configured: `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_SUBSCRIPTION_ID`
- GitHub variables configured: `ACR_NAME`, `ACR_LOGIN_SERVER`, `AKS_RESOURCE_GROUP`, `AKS_CLUSTER_NAME`
- Terraform infrastructure has been applied (AKS, ACR, Key Vault, SQL)

### Frontend Deployment
1. Create a PR touching `frontend/**` — CI runs build + test + Trivy scan
2. Merge PR to `main` — `frontend-cd.yml` triggers
3. Approval gate: a required reviewer must approve in the `production` environment
4. Pipeline builds image tagged with git SHA, pushes to ACR, runs `helm upgrade`
5. Rollout status check fails the pipeline if probes don't pass within 5 minutes

### Backend Deployment
Same flow as frontend, but with `backend/**` path filter and `backend-cd.yml`.

### Manual Deployment (emergency)
```bash
# Get AKS credentials
az aks get-credentials --resource-group rg-resumeats-prod --name aks-resumeats-prod

# Deploy frontend
helm upgrade --install frontend ./k8s/helm/frontend \
  --namespace resumeats-prod \
  -f k8s/helm/frontend/values-production.yaml \
  --set image.repository=acrresumeatsprod.azurecr.io/frontend \
  --set image.tag=<GIT_SHA> \
  --set secretProviderClass.tenantId=<TENANT_ID>

# Deploy backend
helm upgrade --install backend ./k8s/helm/backend \
  --namespace resumeats-prod \
  -f k8s/helm/backend/values-production.yaml \
  --set image.repository=acrresumeatsprod.azurecr.io/backend \
  --set image.tag=<GIT_SHA> \
  --set secretProviderClass.tenantId=<TENANT_ID>
```

## Rollback

### Helm Rollback (most recent release)
```bash
helm rollback frontend 1 -n resumeats-prod
helm rollback backend 1 -n resumeats-prod
```

### Rollback to a Specific Image Tag
```bash
# Find the previous image tag from ACR
az acr repository show-tags --name acrresumeatsprod --repository frontend --orderby time_desc --top 10

# Redeploy with the previous tag
helm upgrade --install frontend ./k8s/helm/frontend \
  --namespace resumeats-prod \
  -f k8s/helm/frontend/values-production.yaml \
  --set image.repository=acrresumeatsprod.azurecr.io/frontend \
  --set image.tag=<PREVIOUS_GIT_SHA>
```

### Kubernetes Rollout Undo (immediate, before Helm rollback)
```bash
kubectl rollout undo deployment/frontend -n resumeats-prod
kubectl rollout undo deployment/backend -n resumeats-prod
```

## Secret Rotation

### SQL Admin Password
The SQL admin password is generated by Terraform and stored in Azure Key Vault. To rotate:

1. **Terraform taint the random password:**
   ```bash
   cd infra/terraform
   terraform taint random_password.sql_admin
   terraform plan    # review the change
   terraform apply   # generates new password, updates SQL server + Key Vault
   ```

2. **Verify the new secret in Key Vault:**
   ```bash
   az keyvault secret show --vault-name kv-resumeats-prod --name sql-admin-password
   ```

3. **Restart pods to pick up the new secret (if CSI driver is used):**
   ```bash
   kubectl rollout restart deployment/backend -n resumeats-prod
   kubectl rollout restart deployment/frontend -n resumeats-prod
   ```

### GitHub OIDC Credentials
If the OIDC App Registration needs to be recreated:
1. `terraform taint azuread_application.github_actions` in the github-oidc module
2. `terraform apply` — creates a new App Registration + federated credential
3. Update GitHub secrets: `AZURE_CLIENT_ID`, `AZURE_TENANT_ID` with the new outputs

## Scaling

### Manual Scaling (immediate)
```bash
kubectl scale deployment backend -n resumeats-prod --replicas=6
kubectl scale deployment frontend -n resumeats-prod --replicas=6
```

### Adjusting HPA Limits
Edit `values-production.yaml` for the relevant chart:
```yaml
hpa:
  minReplicas: 3
  maxReplicas: 10  # increase from 6
  targetCPUUtilizationPercentage: 60  # scale earlier
```
Then: `helm upgrade frontend ./k8s/helm/frontend -n resumeats-prod -f k8s/helm/frontend/values-production.yaml ...`

### Node Pool Autoscaling
The AKS user node pool autoscales between 2–4 nodes. To change:
```bash
az aks nodepool update \
  --resource-group rg-resumeats-prod \
  --cluster-name aks-resumeats-prod \
  --name user \
  --min-count 2 \
  --max-count 6
```

## Incident Response

### Severity Levels
- **SEV1 (Critical):** Complete outage, data loss, security breach
- **SEV2 (High):** Degraded service, high error rate, partial outage
- **SEV3 (Medium):** Minor degradation, alerts firing but service functional

### Response Steps
1. **Acknowledge** the alert (PagerDuty/Slack)
2. **Assess** severity and impact
3. **Communicate** to stakeholders
4. **Investigate** using dashboards and logs
5. **Mitigate** (rollback, scale, restart)
6. **Resolve** and verify
7. **Post-mortem** within 48 hours for SEV1/SEV2

### Useful Commands
```bash
# Check pod status
kubectl get pods -n resumeats-prod

# Check pod logs
kubectl logs -n resumeats-prod -l app.kubernetes.io/name=backend --tail=100

# Describe a problematic pod
kubectl describe pod <POD_NAME> -n resumeats-prod

# Check events
kubectl get events -n resumeats-prod --sort-by='.lastTimestamp'

# Check HPA status
kubectl get hpa -n resumeats-prod

# Check node status
kubectl get nodes -o wide
kubectl describe node <NODE_NAME>
```

## Alert Runbook Entries

### HighErrorRate
- **Trigger:** 5xx error rate > 5% for 5 minutes
- **Action:** Check backend logs for exceptions, check if OCR is failing, check if pods are OOM-killed
- **Mitigation:** Rollback to previous image if a recent deployment caused it; scale up if overloaded

### ElevatedLatency
- **Trigger:** p95 latency > 300ms for 10 minutes
- **Action:** Check CPU/memory usage, check if OCR is being triggered on scanned PDFs, check node load
- **Mitigation:** Scale up backend pods, increase CPU limits, check for slow queries (if SQL is used)

### CrashLoopBackOff
- **Trigger:** Pod restarted > 5 times in 1 hour
- **Action:** `kubectl describe pod <name>` and `kubectl logs <name> --previous`
- **Mitigation:** Rollback to previous image; check if a config/secret change broke startup

### PodUnschedulable
- **Trigger:** Pod unschedulable for > 10 minutes
- **Action:** Check node capacity, check resource requests vs available node resources
- **Mitigation:** Increase node pool max count, reduce pod resource requests, or manually add nodes

### HighNodeCPU / HighNodeMemory
- **Trigger:** Node resource usage > 85% for 15 minutes
- **Action:** Check what pods are consuming resources, check if autoscaler is keeping up
- **Mitigation:** Increase node pool max count, optimize pod resource requests
